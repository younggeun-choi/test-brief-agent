## 설정

- 수신 대상: `#ai-brief`

## 작업 순서

1. `summary/` 폴더에서 가장 최근에 추가된 `YYYY-MM-DD.md` 파일을 찾아 읽어라.
2. Slack 커넥터로 다음 메시지를 전송하라:
   - 대상: 위 "수신 대상"
   - 메시지 첫 줄(타이틀): `*[일일 브리핑] YYYY-MM-DD*` (Slack 굵게 표기)
   - 메시지 본문: summary 파일 내용 전체를 그대로 (마크다운 형식 유지)
3. 전송 성공 응답(메시지 ID 또는 타임스탬프)을 확인한 뒤 종료하라.

## 참고 — Slack 마크다운 특성

- 일반 마크다운(`#`, `##` 헤더, `**굵게**`, `[링크](url)`)은 Slack에서 일부만 렌더링됩니다.
- Slack의 굵게는 `*굵게*` (별표 하나), 링크는 `<url|텍스트>` 형식.
- 본 스킬은 변환 없이 원문을 그대로 전송 — Slack에서 일부 형식이 풀어진 채로 보이더라도 정상.
- 원본 마크다운은 GitHub `summary/` 폴더에 그대로 보관되므로 보존 손실 없음.

## 금지 사항

- GitHub에 추가 커밋 금지 (relay는 읽기·전송만 담당)
- 메시지 본문 임의 수정·요약·재작성 금지
- summary 파일 수정·삭제 금지
